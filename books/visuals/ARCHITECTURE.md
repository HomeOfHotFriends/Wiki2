# Visual Integration Architecture for /books

## Purpose
This document outlines the structure and approach for integrating visual content (images, diagrams, motifs) into the `/books` directory. It ensures visuals reinforce esoteric, generative, and recursive teaching, and provides a foundation for future visual expansion.

## Directory Structure
- `/books/visuals/` — Central repository for all visual assets (SVG, PNG, JPG, Mermaid diagrams, etc.)
- Visuals are referenced from book HTML/Markdown files using relative paths.
- Each visual asset should have a clear, descriptive filename and a short README entry.

## Visual Motifs & Integration
- Visuals should reinforce whakapapa (genealogy), recursion, and generative/esoteric themes.
- Use lateral and genealogical diagrams, recursive glyphs, and symbolic motifs.
- Prefer SVG or Mermaid for diagrams to allow easy editing and versioning.

## Documentation
- All new or changed visuals must be documented in `/books/visuals/README.md` with:
  - Filename
  - Description
  - Associated book(s)
  - Date added/changed

## Example Visual Motifs
- Rhizome lattice (recursive, branching)
- Whakapapa spiral (genealogical layering)
- Generative glyphs (emergent forms)
- Lateral bridges (cross-book connections)
